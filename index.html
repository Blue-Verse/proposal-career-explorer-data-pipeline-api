핵심 요구사항 분석

  • 데이터 파이프라인(ETL) — 고용통계·커리어넷·워크넷 등 공공 Open API 연동, 일일 호출 제한(Rate Limit)을 고려한 안정적 수집 로직, Raw 데이터 추출·변환·적재, 주기적 자동 업데이트 배치/스케줄러
  • 데이터베이스 설계 — 채용·직업군·자격증·시험 합격률 간 관계형(RDBMS) 스키마 설계, 검색·응답 속도를 위한 인덱스 설계, 향후 데이터 확장을 고려한 테이블 구조
  • 백엔드 API 서버 — 내부 프론트엔드 연동용 RESTful API, 직군 선택 시 관련 자격증·시험 합격률 반환 조회 API, 테스트 계정 기반 기초 인증, 상세 API 명세서(Swagger/Postman)
  • LLM/RAG 맞춤형 보고서(선택 과업, 본 제안 포함) — 적재된 직무 데이터를 컨텍스트로 활용하는 RAG 파이프라인, OpenAI 연동 맞춤형 경력 탐색 텍스트 보고서 생성, 성향·직군 기반 프롬프트 엔지니어링
  • 산출물 — 백엔드·파이프라인 전체 소스코드, DB 설계 문서(ERD), API 명세서(Swagger/Postman)

제안 포인트

1. 공공 데이터 수집의 핵심 리스크 — Rate Limit과 안정성 — 을 먼저 설계합니다. 다수 외부 API를 상용 서비스에서 안정적으로 연동·운영하고, 가상계좌·KYC 등 외부 연동을 크론 기반 배치로 자동화한 경험을 보유하고 있습니다. 공공 API별 호출 쿼터 관리, 지수 백오프 재시도, 실패 복구·중복 적재 방지를 수집 로직 설계 단계에서부터 반영합니다.

2. 예산 우선순위에 맞춰 핵심 기능부터 단계적으로 납품합니다. 공고의 우선순위(파이프라인 → DB → API → LLM)를 그대로 마일스톤으로 구성하여, 각 단계가 독립적으로 검수·활용 가능하도록 설계합니다. LLM 과업은 본 제안에 포함하되, 기초 파이프라인이 항상 우선 완성되도록 일정을 배치합니다.

3. RAG 기반 맞춤형 보고서까지 예산 내 포함 제안합니다. ChatGPT/OpenAI 연동 AI 보고서 생성과 RAG 파이프라인을 실제 서비스에 구축·운영한 경험을 바탕으로, 적재된 직무 데이터를 컨텍스트로 활용하는 보고서 생성 로직을 함께 제공합니다.

4. 프론트엔드 자체 개발을 전제로, 연동 친화적인 API와 문서를 제공합니다. 내부 프론트팀이 바로 활용할 수 있도록 일관된 응답 스키마, 페이지네이션·필터 규칙, Swagger 자동 문서화, 샘플 호출 예제를 함께 산출합니다.

5. 정부지원사업 행정 절차에 대한 이해를 갖추고 있습니다. 사업자등록증 보유로 세금계산서 발행이 가능하며, 산출물 검수·증빙 자료 제출 등 정부지원 과제 정산 절차에 맞춰 문서를 정리합니다.

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 기능 범위를 명확히 합니다.

1. 프로젝트 배경 및 목적

사용자가 원하는 직군을 선택하면 해당 직무와 관련된 자격증 정보, 시험 합격률 등의 데이터를 제공하는 맞춤형 경력 탐색 서비스를 준비하고 있습니다. 본 과업은 경산시 청년 예비 창업가 육성 사업의 지원을 받아 진행되며, 프론트엔드 및 화면단은 내부에서 자체 개발합니다. 따라서 본 프로젝트는 외부 공공 데이터를 안정적으로 수집하는 파이프라인 구축과 프론트엔드와 통신할 수 있는 백엔드 API 서버 개발에 집중하며, 제한된 예산 안에서 핵심 기능 구현을 최우선으로 합니다.

3. 기능 요구사항

3.1 데이터 파이프라인 (ETL)

  • [필수] 고용통계·커리어넷·워크넷 등 공공 Open API 연동 및 데이터 수집 로직
  • [필수] API별 일일 호출 제한(Rate Limit)을 고려한 안정적 수집 (호출 쿼터 관리, 백오프·재시도)
  • [필수] Raw 데이터에서 서비스 필요 속성만 추출·변환하여 DB 적재 (ETL)
  • [필수] 데이터 최신화를 위한 주기적 자동 업데이트 배치/스케줄러

3.2 데이터베이스 설계

  • [필수] 채용·직업군·자격증·시험 합격률 간 관계형(RDBMS) 스키마 설계
  • [필수] 검색·응답 속도를 고려한 인덱스 설계
  • [필수] 향후 서비스 고도화·데이터 확장을 고려한 테이블 구조

3.3 백엔드 API 서버

  • [필수] 내부 프론트엔드 연동용 RESTful API 서버
  • [필수] 직군 선택 시 관련 자격증 정보·시험 합격률 반환 조회 API
  • [필수] 테스트용 계정 기반 기초 인증 (복잡한 인증/인가 로직 제외)
  • [필수] 상세 API 명세서 (Swagger / Postman)

3.4 LLM/RAG 맞춤형 보고서 (선택 과업 — 본 제안 포함)

  • [포함] 적재된 직무 데이터를 컨텍스트로 활용하는 RAG 기반 파이프라인
  • [포함] OpenAI API 연동 맞춤형 경력 탐색 텍스트 보고서 생성
  • [포함] 사용자 성향·선택 직군에 맞춘 프롬프트 엔지니어링

4. 비기능 요구사항

항목요구사항
안정성공공 API 장애·Rate Limit 초과 시에도 수집 파이프라인이 중단되지 않도록 재시도·실패 격리 처리
데이터 정합성중복 적재 방지(Upsert), 수집 단위 트랜잭션, 최신화 시점 기록
성능주요 조회 API 응답 시간 최적화 (인덱스 기반 쿼리, 페이지네이션)
보안HTTPS, 환경변수 기반 비밀키 관리, 테스트 계정 인증, SQL Injection 방어(ORM)
확장성데이터 소스·직군 추가 시 커넥터 단위로 확장 가능한 구조
문서화Swagger 자동 문서, ERD, 배포 가이드 제공

5. 기술 제약사항

  • 프론트엔드 및 화면단은 클라이언트 내부 자체 개발 (본 과업 범위 외)
  • 언어·프레임워크는 개발사 제안 — 데이터 처리·AI 연동에 강점이 있는 Python/FastAPI 스택 제안
  • 공공 API의 키 발급·이용 약관·호출 한도는 클라이언트 명의 발급 키 기준으로 협의 필요
  • 제한된 예산 내 핵심 기능 우선 — 마일스톤 단위 단계 납품

6. 범위 외 사항 (Out of Scope)

  • 프론트엔드 / UI 화면 개발 (내부 자체 개발)
  • 복잡한 회원 인증·인가, 권한 관리 체계 (기초 테스트 인증으로 대체)
  • 결제·정산 등 비즈니스 트랜잭션 기능
  • 외부 유료 API 사용료 및 클라우드 인프라 비용

일정 제안

총 90일 기준, 공고의 우선순위(파이프라인 → DB → API → LLM)에 맞춘 단계별 마일스톤 및 산출물을 제안합니다.
